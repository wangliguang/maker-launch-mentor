# 硬件理解确认与 Skill 精简设计

## 背景

当前 `SKILL.md` 在 `HARD-GATE`、Anti-Patterns、Checklist、流程图、工作流程、强制规则、Gotchas、输出风格和 Key Principles 中多次重复相同约束。重复有助于强调关键行为，但也让提示词偏长，增加维护成本，并可能因多处表述不一致而产生歧义。

本次调整同时增加一个教学门槛：介绍项目模块所需硬件时，必须逐个确认孩子是否了解；孩子第一次表示不了解，就立即让家长参与。

## 目标

1. 为每个模块增加逐个硬件的理解确认流程。
2. 明确普通硬件理解确认与危险工具安全门的区别。
3. 在不丢失现有教学、安全、代码和排错约束的前提下明显缩短 `SKILL.md`。
4. 让每条规则只在一个主要位置完整定义，关键门槛只做简短重申。

## 不在范围内

- 不修改项目说明、完整代码或硬件清单。
- 不改变三个游戏规则检查、危险工具安全门和逐模块累加代码等现有行为。
- 不发布新版本，不打 Git tag，也不覆盖全局安装；这些操作仍需用户确认修改完成后再进行。
- 不要求孩子在家长讲解后通过额外答题。

## 文档结构

精简后的 `SKILL.md` 保留六个主体：

1. 定位与目标。
2. `HARD-GATE`：只保留会阻止流程继续的游戏规则、硬件理解、危险工具、模块顺序、代码读取和先完成再扩展规则。
3. 行动前必须读取的资料。
4. 教学主流程：认识学员、选择项目、理解规则、检查硬件、逐个认识硬件、按模块制作、验证和复盘。
5. 专项规则：危险工具、电路类比和逐步累加代码。
6. 输出风格与项目特有 Gotchas。

删除完整流程图、重复 Checklist、两组重复的强制规则和 Key Principles。删除前先把其中独有约束合并进主流程或专项规则。

## 硬件理解确认流程

进入项目模块的 `所需硬件` 步骤后，对该模块的硬件逐个执行：

1. 一次只介绍一个硬件，并让孩子观察、触摸或指出它。
2. 介绍完当前硬件后，询问孩子是否了解这个硬件。
3. 孩子表示了解时，继续介绍下一个硬件。
4. 孩子表示不了解、不会辨认或明显困惑时，立即请家长参与，并暂停介绍后续硬件以及接线、代码等后续步骤。
5. 家长到场后，由家长决定何时继续；AI 不设置额外答题门槛。
6. 当前模块的全部硬件都确认完毕后，才能进入 `接线方式`。

这是普通硬件的理解门槛。家长允许继续后，不要求家长在普通制作步骤中全程陪同。

危险工具继续使用独立且更严格的安全门：家长必须到场并全程陪同，由家长手把手指导；AI 不提供危险工具的使用教程、动作分解或独立操作指导。

缺少硬件时继续执行原有规则：说明缺少的硬件及用途，提醒家长准备，并暂停实物制作、接线、焊接和上传代码。

## 必须保留的现有行为

- 三条游戏规则全部理解后才能检查硬件和进入制作。
- 项目介绍阶段只介绍玩法，不提前讲硬件、难度或专业术语。
- 每个模块按项目文档的硬件、接线、代码、验证和排错顺序推进。
- 只有进入当前模块的 `编写代码` 步骤后，才能读取完整代码文件。
- 代码按模块逐步累加，最后一个模块完成时才形成完整项目代码。
- 缺少硬件时暂停实物制作。
- 危险工具必须由家长到场、全程陪同并手把手指导。
- 面向孩子时一次只推进一件事、一次只问一个问题，回复通常不超过三个短句。
- 排错时先问现象；可能是接线问题时先检查接线，再修改代码。
- 当前项目完成前不增加扩展需求。

## 异常与暂停条件

- **孩子不了解普通硬件**：暂停后续流程，请家长参与；由家长决定何时恢复。
- **家长尚未参与**：保持暂停，不继续介绍后续硬件、接线或代码。
- **硬件缺失**：暂停全部实物制作步骤，允许继续做不依赖实物的简单概念说明。
- **涉及危险工具且家长未全程陪同**：暂停危险工具步骤，不输出具体操作指导。
- **模块验证失败**：先收集现象，再按项目文档排错；接线相关问题优先检查硬件连接。

## 验证方案

1. 对照修改前的原文逐项检查，确认独有约束均已迁移，没有因删除重复章节而丢失。
2. 检查普通硬件理解门槛与危险工具安全门的措辞，确保家长参与程度不会混淆。
3. 用四种情形检查新增流程：孩子了解、孩子不了解、家长未参与、家长允许继续。
4. 比较修改前后的行数和重复关键词，确认篇幅与重复度明显下降。
5. 检查 Markdown 标题、列表、标签和文件路径引用。

## 验收标准

- 每个硬件介绍后都有一次独立理解确认。
- 孩子第一次表示不了解时立即请家长参与并暂停流程。
- 家长决定何时恢复，孩子无需再通过额外答题。
- 普通硬件理解门槛不会削弱危险工具安全门。
- 原有核心教学、安全、模块、代码和排错行为全部保留。
- `SKILL.md` 的结构更短，完整规则不再在多个章节重复展开。
